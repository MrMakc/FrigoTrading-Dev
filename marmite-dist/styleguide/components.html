Banner messages
JS Options
{
  project: 'projectJsName',
  hideJs: true, // if hideJs == true, hideClass is note needed
  hideClass: 'hide-banner', // class to animate the hide if hideJs = false
  hiddenClass: 'as--hidden', // class to remove when banner is hidden by default
  caping: 0,
  container: '.c-banner-messages',
  availableDuration: 13,
}

HTML Options 
{
  banner: 'bannerName',
  hideJs: true,
  hideClass: 'hide-banner',
  hiddenClass: 'as--hidden',
  caping: 3,
  availableDuration: 13,
}
Usage

If you click on the consent button .js-banner-btn-accept :
- the banner close
- the cookie `projectJsName``bannerName` is create
- the the banner won't display again for 13 month duration.

If you click on the close button .js-banner-btn-close :
- the banner close
- the sessionStorage item `projectJsName``bannerName` is create
- the banner won't display until you close your session.

If you don't click on these two button and don't set caping option :
- the banner will still display if you reload or change page.

If you don't click on these two button and set caping option (n > 0) :
- the banner will display n times if you reload or change page.
- THEN, the banner won't display any more until you close your session.

The button .sg-banner-messages-reset is only used in the styleguide.
Never use it in your website.

Images Size

Available image size
  • exemple (500x500)
Images are in bg cover

You have to set the figure[data-pict] dimensions in css and the image will always be covered.

The img is crop with the css property object-fit


Options
  • [data-pict="top"] (force align top)
  • [data-pict="bottom"] (force align bottom)
  • [data-pict="left"] (force align left)
  • [data-pict="right"] (force align right)
  • [data-pict="bg-contain"] (force bg size contain)
  • [data-pict="default"] (restore image default size)
